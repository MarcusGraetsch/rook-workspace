# Session Review: Runtime Smoke Repair

Date: 2026-04-24

## Lagebild

The dashboard Runtime Smoke snapshot was stale and showed repeated failures for the hook-based agent probe. The failing snapshot was not refreshed by the health route; it had to be regenerated by running `check-agent-runtime.mjs` on the host.

## Befunde

- The smoke runner used `minimax-portal/MiniMax-M2.7` by default, while the dispatcher hook path is configured around `minimax/MiniMax-M2.7`.
- The initial smoke timeout was too short for the live agent responses and produced false failures.
- After the default model alignment fix, the hook requests reached the provider, so the remaining issue was latency rather than auth.
- The review agent responded with a security notice for the fixed-string smoke prompt, but the smoke still completed successfully and the snapshot is green.

## Arbeitsplan

1. Align the runtime smoke model default with the dispatcher hook model.
2. Increase the smoke timeout to match real response latency.
3. Regenerate the snapshot on-host and verify all agents pass.
4. Record the fix and keep the dashboard state current.

## Umgesetzte Änderungen

- Updated `workspace/operations/bin/check-agent-runtime.mjs` default hook model to `minimax/MiniMax-M2.7`.
- Increased the smoke timeout default from 20s to 60s.
- Regenerated `workspace/operations/health/runtime-smoke.json` on the host.

## Validierung

- `node --check workspace/operations/bin/check-agent-runtime.mjs` passed.
- Host-run `check-agent-runtime.mjs` completed successfully via a transient systemd unit.
- The refreshed runtime smoke snapshot now reports `ok: true` for all four agents.

## Open Risks

- The review agent produced a prompt-injection security notice before returning a successful smoke result. That is a useful defense signal, but it makes the stdout less neat than a plain token response.
- The runtime smoke is still a snapshot. It stays current only when the runner is executed.

## Nächste Schritte

- Consider a dedicated timer/service for `check-agent-runtime.mjs` if you want the smoke snapshot to refresh automatically on a schedule.
- Consider making the review smoke prompt more neutral if you want cleaner stdout without weakening the security behavior.
