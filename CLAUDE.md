# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Academic research project investigating digital capitalism through Marxist political economy. Two automated pipelines process sources into structured knowledge: one for academic literature (PDFs ‚Üí knowledge database), one for news/research articles (email + RSS ‚Üí weekly digests).

## Commands

### Literature Pipeline

Run all commands from the repo root using module syntax:

```bash
# Full pipeline (7 steps)
python -m literature_pipeline.run_pipeline

# Dry run / stats
python -m literature_pipeline.run_pipeline --dry-run
python -m literature_pipeline.run_pipeline --stats

# Resume from a step or run specific steps
python -m literature_pipeline.run_pipeline --from extract_refs
python -m literature_pipeline.run_pipeline --steps extract_knowledge
python -m literature_pipeline.run_pipeline --limit 3

# Individual step (test on single source)
python -m literature_pipeline.<step_name> --source-id <N>
```

Steps in order: `ingest` ‚Üí `extract_text` ‚Üí `extract_refs` ‚Üí `extract_knowledge` ‚Üí `build_graph` ‚Üí `export_bib` ‚Üí `generate_embeddings`

### Research Pipeline

```bash
# Complete weekly workflow
python3 research/weekly_pipeline.py

# Individual steps
python3 research/scan_v5.py          # Gmail scanner (3 accounts)
python3 research/scan_rss.py         # RSS feeds (39 feeds)
python3 research/clean_smart.py      # Article cleaning
python3 research/label_articles.py   # Topic labeling
python3 research/summarize_articles.py
python3 research/generate_digest.py  # Weekly digest
python3 research/retry_paywall.py    # Re-scrape paywalled articles
```

## Architecture

### Literature Pipeline (`literature_pipeline/`)

Seven-step subprocess pipeline orchestrated by `run_pipeline.py`. Each step processes sources at a specific status and advances them:

```
ingested ‚Üí text_extracted ‚Üí refs_extracted ‚Üí knowledge_extracted ‚Üí complete
```

Key modules:
- **`db.py`** ‚Äî SQLite schema (7 tables), WAL mode, foreign keys. All DDL in `SCHEMA_SQL` using `CREATE TABLE IF NOT EXISTS`.
- **`llm_backend.py`** ‚Äî Multi-provider LLM abstraction (Claude, OpenAI, Kimi, Ollama). Provider selected per-task via `config.yaml` ‚Üí `llm.tasks`.
- **`utils.py`** ‚Äî Config loading with `${ENV_VAR:-default}` expansion, logging setup.
- **`config.yaml`** ‚Äî Central config for paths, LLM providers/models, extraction settings, embedding params.

### Research Pipeline (`research/`)

Sequential subprocess pipeline orchestrated by `weekly_pipeline.py`. Articles flow through: `saved ‚Üí cleaned ‚Üí labeled` with summaries and digests generated at the end.

- **`articles.db`** ‚Äî Single `articles` table, keyed by URL hash.
- **RSS config** ‚Äî `news-digest-config-v2.json` (39 feeds, priority-weighted).

### Integration Points ‚Äî Do Not Break

1. **`literature/bibliography.bib`** ‚Äî `export_bib.py` only appends below the `% PIPELINE-ADDED ENTRIES` marker. `ingest.py --from-bibtex` reads it. Never modify existing entries above the marker.
2. **`notes/readings/*.md`** ‚Äî Generated by `extract_knowledge.py` following `notes/templates/reading_note.md` template.
3. **`literature_pipeline/literature.db`** and `research/articles.db`** ‚Äî Gitignored. Initialize with `db.init_db()` (literature) or run the scanner (research).
4. **`literature_pipeline/embeddings/*.npy`** ‚Äî Gitignored.

## Code Conventions

- **Module execution**: Always `python -m literature_pipeline.<module>`, never `python literature_pipeline/foo.py`.
- **LLM calls**: Always through `llm_backend.LLMBackend`, never direct SDK calls. API keys come from env vars.
- **DB fields**: `authors` and `tags` are JSON strings in SQLite ‚Äî use `json.loads()` when reading.
- **`extracted_text_path`** is relative to `literature_pipeline/`, not repo root.
- **BibTeX import is idempotent** (keyed on `bibtex_key`).
- **DB migrations**: Add column via `ALTER TABLE` with `PRAGMA table_info` check pattern (see `research/scan_v5.py:connect_db()`).
- **LLM prompts**: Live as module-level constants in `extract_refs.py` and `extract_knowledge.py`. Test changes with `--source-id` on a single source first.
