# Enterprise Architecture — TOGAF & ADOIT

## Überblick

TOGAF, ADOIT, EA als Beratungsdisziplin.

## TOGAF

**The Open Group Architecture Framework**

| Component | Beschreibung |
|-----------|-------------|
| Preliminary | Strategie, Governance, Architektur-Board |
| Architecture Vision | Stakeholder Concerns, Vision |
| Business Architecture | Geschäftsprozesse, Organisation |
| Data Architecture | Datenmodelle, Datenflüsse |
| Application Architecture | Applikationen, Integration |
| Technology Architecture | Infrastruktur, Plattformen |
| Opportunities & Solutions | Roadmap erstellen |
| Migration Planning | Transition Architektur |

## Typische EA-Artefakte

| Artefakt | Beschreibung |
|----------|-------------|
| Architektur-Prinzipien | Leitlinien für Entscheidungen |
| Capability Map | Was kann das Unternehmen? |
| Applikations-Landschaft | Alle Applikationen + Abhängigkeiten |
| Datenfluss-Diagramm | Wie fließen Daten? |
| Target Architecture | Zielarchitektur (Future State) |
| Transitionsarchitektur | Weg dorthin |
