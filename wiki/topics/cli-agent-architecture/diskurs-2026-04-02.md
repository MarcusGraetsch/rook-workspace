## Kernthema

**Technische Migration von KI-Agenten: Weg von starren MCP-Schemata, hin zu CLI-basierten Architekturen.**

Das "Warum" (Tokenersparnis, Latenz, Kosteneffizienz) wird bewusst ausgeschlossen. Fokus: Das "Wie" — wie zähmt man die rohe Kommandozeile für autonome Agenten?

---

## Zwei Philosophien

### Position A: Statische Kapselung (Mann)
- **Thin Wrappers** — kurze Python/PowerShell-Skripte kapseln komplexe Logik
- **JSON-erzwungene CLI-Tools** — `output=json` Flags fest verdrahten
- **Skill.md als starres Gesetzbuch** — exakte Parameter definiert (kein Interpretationsspielraum)
- **Vorteile:** Vorhersehbar, auditierbar, deterministisch, sicher für regulierte Umgebungen (Banken, Healthcare)
- **Kritik an Position B:** "Hope-Based Engineering"

### Position B: Native CLI-Orchestrierung (Frau)
- **Dynamische Just-in-Time-Discovery** — Agent liest `help` zur Laufzeit, baut Befehl autonom
- **Unix-Pipelines über Process Builder** — schwere Datenverarbeitung auf OS-Ebene, LLM hält Kontext frei
- **Skill.md als Handbuch mit Profi-Tipps** — semantisches Priming, keine starren Befehle
- **Vorteile:** Flexibel, evolutionär-resilient, Selbstheilung bei Tool-Updates
- **Kritik an Position A:** "Rückschritt in die 2010er Denkweise"

---

## Schlüsselkonzepte

### 1. Der 800-Token-Trick
Tool-Definitionen auf ca. 800 Tokens komprimieren → bleibt im Aufmerksamkeitszentrum des LLMs. Kein "Lost-in-the-Middle" bei 10.000-Token-JSON-Schemata.

### 2. Progressive Offenbarung (Progressive Disclosure)
Nicht alles beim Start laden, sondern erst im Moment der Ausführung. JIT-Dokumentation.

### 3. Stille Fehler in Pipelines
Das Kernproblem: `stderr` wird bei Unix-Pipes in separaten Stream geschrieben.Wenn `FilterByTier` abstürzt → `CountByTier` empfängt leere Menge → gibt "0" aus → LLM weiß nicht dass ein Fehler passiert ist.

**Lösung Position A:** JSON-Verträge mit Status-Codes
**Lösung Position B:** Stderr am Ende bündeln und dem Modell als Systemnachricht zeigen

### 4. Self-Healing Agents
Agent führt Help aus → erkennt Syntaxänderung → passt Befehl autonom an → führt erfolgreich aus.
**Kritik:** In regulierten Umgebungen ist Abbruch ein Feature, kein Bug.

### 5. Thin Wrappers vs Raw CLI
- **Wrapper:** `getNonCompliantDevices.ps1` — kapselt Logik, Agent drückt nur auf "Knopf"
- **Raw:** Agent orchestriert 5 separate CLI-Calls selbst

### 6. Stdout/Stderr Pipeline
Daten fließen über CSTDOUT des ersten Tools direkt in STDIN des zweiten — LLM sieht nur Endergebnis, nicht Zwischendaten.

## Konvergenzpunkt

**Die Markdown-Datei (skill.md) ist das neue zentrale Steuerungselement** — ersetzt die schwerfälligen JSON-Verträge der MCP-Ära.

Unabhängig davon ob statisch oder dynamisch:
- Abkehr vom Vorabladen gigantischer Schemata
- Progressive Offenbarung als architektonischer Schlüssel dieses Jahrzehnts
