## Beratungs-Themen

### Cloud Exit Strategie
1. **Ist-Analyse** — Cloud-Dienste, Vendor Lock-in Grad, Datenklassifizierung
2. **Exit-Optionen** — Cloud → On-Prem, Cloud → andere Cloud, Hybrid
3. **Exit-Kosten** — Datenmigration, Neuentwicklung, Schulungen
4. **Empfehlung** — Business Case erstellen, Roadmap

### Cloud Readiness Assessment
```
Phase 1: Assessment (2-4 Wochen) — Infrastruktur audit, Applikationsanalyse, Skill-Evaluation
Phase 2: Strategie (2-3 Wochen) — Zielarchitektur, Migration-Ansatz, Roadmap
Phase 3: Pilot (4-6 Wochen) — 1-2 Applikationen migrieren, Learnings
Phase 4: Rollout (3-12 Monate) — Stufenweise Migration, Schulungen
```
